Multi-Device Synchronization of DSA Analog Input and Output Using a Shared Timebase & Trigger with DAQmx

VERSION 2 
Created on: May 20, 2010 12:30 PM by Verne_D - Last Modified:  May 29, 2014 8:51 PM by Grandmaster_P


Description:

This example configures a total of four tasks on two PXI Dynamic Signal Acquisition (DSA) devices. Each device runs a task for analog input and another for analog output. This VI illustrates continuous analog input and output operations.

Instructions for Running:

1. Select which type of Synchronization method you want to use.
2. Select the physical channels to corresponding to the signal connection on each device.
3. Enter the minimum and maximum expected voltage for the input and output operations on each device.
     Note: For optimal accuracy, match the input range to the expected voltage level of the measured signal (for input) and generated signal (for output).
4. Set the number of samples to acquire per channel. This parameter determines how many points will be read and generated with each iteration of the WHILE loop.
5. Set the rate of the acquisition and generation. The same sampling rate is employed for input and output operations on each device.
     Note: The sampling rate should be at least 2.2 times the maximum frequency component of the signal being acquired on analog input and the signal being generated on analog output. Frequency components beyond about 0.47 times the sampling rate will be eliminated by the alias and imaging protection on the DSA device.

Block Diagram Steps:

1. Create an analog input and output channel for both the master and slave devices.
2. Set timing parameters for a continuous acquisition. The sample rate and block size are set to the same values for each device.
3. There are two types of synchronization available on DSA devices. The first method shares the Sample Clock Timebase across the PXI_Star bus. It also uses the Sync Pulse which is shared across the PXI_Trig / RTSI bus. The second method uses the Sync Pulse in conjunction with the PXI Reference clock 10 on the PXI backplane. 
4. Call the Get Terminal Name with Device Prefix VI to append the device name to the signal name for the acquisition start trigger. This signal is then used as an output generation trigger on the master as well as an acquisition start trigger and generation start trigger on the slave. The signal is shared along the PXI_Trig / RTSI bus.
5. Synthesize a standard waveform (sine, square, or triangle) and load this data into the output RAM buffer for each device.
6. Call the Start VI to start the acquisition and generation tasks.

     Note: The flat case structure provides flow control so that the slave device is armed before the master so that the slave device does not miss the trigger.
7. Read all of the waveform data from both devices.
8. Check the status of the analog output tasks on both devices.
9. Call the Clear Task VI to stop the acquisition and generation and clear the task.
10. Use the popup dialog box to display an error if any.

I/O Connections Overview:

Make sure your signal input and output terminals match the Physical Channel I/O controls.

It is important to ensure that your PXI chassis has been properly configured in NI MAX. If your chassis has not been configured in software before running the VI, your devices may fail to synchronize properly.
